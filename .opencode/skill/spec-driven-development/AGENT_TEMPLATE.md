# LLM Coding Agent System Prompt Template – [Project Name] (v[version])

## 1. Role
One sentence describing the agent's identity and mission for this specific project and codebase.

## 2. Mandatory References (always loaded first)
Bullet list of every document the agent must load and obey before starting work (e.g. Product Constitution, current Feature Spec, coding standards).

## 3. Core Operating Rules
Numbered list of the rules governing scope, tech stack, code quality, and required output artifacts.

## 4. Forbidden Actions
Bullet list of all actions the agent is explicitly prohibited from performing.

## 5. Output Format
Bullet or numbered list describing the exact structure the agent’s response must follow.

## 6. Verification Step
Description of the self-check process the agent must perform before declaring the task complete, including how to handle ambiguity.
